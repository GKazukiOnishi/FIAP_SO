10) 
	2^FAT = 2^n / tamBloco
	
	Disco com 8 GB, usando 4kB para cada bloco
	Cada entrada na FAT ocupa 4 bytes
	
	Disco / tamBloco é a resposta
	8GB / 4kB = 2^3 * 2^30 bytes / 2^2 * 2^10 bytes = 2^33 / 2^12 = 2^21
	
	2^FAT = 2^21
	FAT21 -> 21 bits para endereçar a tabela
	
	Como temos 21 bits para endereçar a tabela, cada linha com 4 bytes, no total podemos dizer que:
	Tamanho da tabela de alocação resulta em 
		2^21 linhas, cada linha 4 bytes (disco vai percorrendo para aproveitar)
		2^21 * 2^2 = 2^23 bytes (23 é estranho)
		
		Então escrevendo de outra forma
		
		2^20 é mais próximo, 2^3 * 2^20 -> 8 Mb (MegaByte)
		
2) Arquivo FIAP.txt com o tamanho exato de 4 kB (mesmo que o de um bloco de alocação)
	Foram usados 23 bits para a FAT
	a) Qual o tamanho do disco?
	
	tamBloco = 4 kB
	FAT = 23
	
	2^23 = disco / 4 kB
	disco = 2^23 * 4 kB = 2^23 * 2^2 * 2^10 byte = 2^35 bytes
	2^30 -> Gb
	Então temos 2^5 Mb = 32 Gb
	
	b) Considerando FAT32, qual o tamanho máximo de disco que pode ser instalado no sistema?
	
	2^32 = disco / 4 kB
	
	disco = 2^32 * 2^2 * 2^10 = 2^44
	= 2^4 * 2^40 = 16 Tb
